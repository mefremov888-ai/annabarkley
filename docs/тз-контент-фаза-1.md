# ТЗ: Контент сайта — фаза 1

**Контекст.** Технически сайт собран (Next.js 14, 40 routes, type-check + build чистые), но контент-блок ICP остаётся плейсхолдерами: у Анны нет фото, нет OG-карточек для соцсетей, у `/blog` есть карточки 5 статей, но самих статей нет. Без этих трёх артефактов запуск невозможен — middle-premium конверсионный сайт без живого образа специалиста и контента не работает.

ТЗ разбито на три независимых под-задачи. Можно делать параллельно.

### A3. Технические требования к фотографу

- Формат: RAW + JPEG (полученные после ретуши)
- Цветовая температура: 4500–5000K (тёплый, не холодный)
- Минимальное разрешение: **3000×2000px** (нужно для retina + crop под OG)
- Обработка: естественная ретушь (без сглаживания кожи в пластмассу)
- Композиция: оставлять «воздух» вокруг лица для будущего crop'а под разные форматы
- Лицензия: полные коммерческие права на использование в течение 5 лет, без указания фотографа на сайте (опционально — в `/credits`)

### A4. Что Анне подготовить к съёмке

### B2. Объём

5 OG-карточек:

| # | Файл | Использование | Заголовок (EN / RU) |
|---|------|---------------|---------------------|
| 1 | `og-default.jpg` | Главная, About, Services | "Anna Barkley — Psychologist in Dubai" / "Анна Баркли — психолог в Дубае" |
| 2 | `og-issues.jpg` | /issues, /issues/[slug] | "What I Help With" / "С чем я помогаю" |
| 3 | `og-pricing.jpg` | /pricing | "Transparent Pricing — From 700 AED" / "Прозрачные цены — от 700 AED" |

| 5 | `og-blog.jpg` | /blog, /blog/[slug] | "Articles on Mental Health" / "Статьи о ментальном здоровье" |

### B3. Технические требования

- Размер: **1200×630px** (стандарт OG)
- Формат: JPG (хорошее сжатие для фото) или PNG (для дизайна с большим текстом)
- Размер файла: ≤ 300КБ
- Цветовое пространство: sRGB

### B4. Дизайн

Шаблон:
- Фон: фотография Анны (для og-default — крупный портрет №1; для остальных — кабинет №3 с blur, или абстрактный градиент `sage-pale → rose-pale`)
- Текст: белый поверх затемнения (rgba(0,0,0,0.35) overlay)
- Шрифт заголовка: Sora 600, размер 60px
- Шрифт подзаголовка: Plus Jakarta Sans 400, размер 28px
- Логотип: фирменный знак + «Anna Barkley» в углу

### B5. Реализация в Next.js

Готовые JPG положить в `web/public/og/og-default.jpg` и т.д. В `metadata` добавить:
```tsx
openGraph: {
  images: [{ url: '/og/og-default.jpg', width: 1200, height: 630 }],
},
twitter: {
  card: 'summary_large_image',
  images: ['/og/og-default.jpg'],
},
```

Для динамической генерации (опционально, фаза 2): использовать `next/og` (`@vercel/og`) — генерирует OG на лету по запросу `/og?slug=X`.

### B6. Что от пользователя

- Готовые фото (см. п. A — №1 и №3 — обязательны)

## C. 5 статей в /blog

### C1. Цель

Перевести `/blog` из «карточки без ссылок» в полноценный раздел с 5 статьями длиной 600–1000 слов каждая. Статьи нужны для:
- SEO (длинный хвост запросов)
- Авторитет / экспертный позиционирование
- Lead magnet («полезный материал — оставьте email чтобы получить ещё»)
- Пища для соцсетей

### C2. Темы (уже зафиксированы в `web/app/blog/page.tsx`)

| # | EN заголовок | RU заголовок | Категория | ICP |
|---|-------------|--------------|-----------|-----|
| 1 | Why Is My Child So Anxious? | Почему мой ребёнок тревожится? | Children | B (родители 6-12) |
| 2 | The Teenage Identity Crisis | Подростковый кризис идентичности | Teens | C (родители 13-17) |
| 3 | Why We Repeat Emotional Patterns | Почему мы повторяем эмоциональные паттерны | Adults | A (взрослые экспаты) |
| 4 | You Don't Have to Be a Perfect Parent | Не нужно быть идеальным родителем | Parenting | B + C |
| 5 | Emotional Challenges of Expat Life | Эмоции экспата | Families | A + B + C |

### C3. Структура каждой статьи

```markdown
# {Заголовок}

{Lede — 1 абзац, 50–80 слов. Хук + обещание чего читатель уйдёт со статьёй.}

## {Подзаголовок 1 — диагноз / контекст}
{2–3 абзаца — что родитель/взрослый видит, как это выглядит со стороны, что часто упускают.}

## {Подзаголовок 2 — почему это так}
{2–3 абзаца — психологическая модель, КПТ/привязанность/нейробиология. Без жаргона. Один аккуратный термин с объяснением.}

## {Подзаголовок 3 — что делать}
{Практические шаги, 3–5 пунктов. Каждый пункт — 1–3 предложения.}

## Когда стоит обратиться
{1 абзац — красные флаги. Не пугать, но и не замазывать.}

> **CTA блок:** Если узнаёте себя — запишитесь на бесплатный 15-минутный звонок. {WhatsApp-кнопка + ссылка на /contact}

---

*Статья носит информационный характер и не заменяет профессиональной консультации.*
```

### C4. Требования к языку и стилю

- **EN:** уровень B2-C1, тон тёплый, профессиональный, без «coach speak» («unlock your potential», «journey» — НЕ).
- **RU:** живой современный, без советизмов и «вы — клиент». Обращение «ты» в тексте недопустимо (мы — лицензированная практика, не блог), но и сухое «Вы» — тоже мимо. Лучше — без местоимений: «Многим знакомо…», «Дети 6–12 часто…».
- **DHA-compliance:** проверить каждую статью по списку запрещённых слов: `cure`, `guaranteed`, `100%`, `permanent fix`, `complete recovery`. Заменять на: `support`, `may improve`, `often helps`, `evidence-based approaches`.
- **Длина:** 600–1000 слов EN. RU чуть короче (русский плотнее).
- **Lectorat:** один читатель, не «семья», не «специалисты». Конкретный человек с конкретной болью.

### C6. Технические доработки кода

#### C6.1. Создать `web/lib/blog.ts`

```ts
export interface BlogPost {
  slug: string;
  cat: { en: string; ru: string };
  bg: string;
  date: string; // ISO
  readingTime: number; // минуты
  t: { en: string; ru: string };
  d: { en: string; ru: string };
  metaDesc: { en: string; ru: string };
  body: { en: string; ru: string }; // markdown или JSX
}

export const POSTS: BlogPost[] = [
  { slug: 'child-anxiety', ... },
  { slug: 'teenager-identity', ... },
  { slug: 'emotional-patterns', ... },
  { slug: 'imperfect-parenting', ... },
  { slug: 'expat-emotions', ... },
];
```

#### C6.2. Создать `web/app/blog/[slug]/page.tsx`

```tsx
import { notFound } from 'next/navigation';
import { POSTS } from '@/lib/blog';
import { BlogArticle } from './BlogArticle';

export function generateStaticParams() {
  return POSTS.map((p) => ({ slug: p.slug }));
}

export function generateMetadata({ params }) {
  const post = POSTS.find((p) => p.slug === params.slug);
  if (!post) return {};
  return {
    title: post.t.en,
    description: post.metaDesc.en,
    alternates: { canonical: `/blog/${post.slug}` },
  };
}

export default function Page({ params }) {
  const post = POSTS.find((p) => p.slug === params.slug);
  if (!post) notFound();
  return <BlogArticle post={post} />;
}
```

#### C6.3. Создать `web/app/blog/[slug]/BlogArticle.tsx`

Client-компонент с `useLang()`, рендерит:
- Хлебные крошки `← Все статьи`
- Заголовок + дата + время чтения
- Категория-tag
- Тело статьи (markdown или JSX с `prose`-классом)
- В конце CTA-блок: WhatsApp + Book free call
- `JSON-LD type=Article` для SEO

#### C6.4. Обновить `web/app/blog/page.tsx`

Заменить «карточки без ссылок» на `<Link href={`/blog/${post.slug}`}>`.

#### C6.5. Обновить `web/app/sitemap.ts`

Добавить:
```ts
const blogRoutes = POSTS.map((p) => ({ path: `/blog/${p.slug}`, priority: 0.7, changeFrequency: 'monthly' as const }));
```

### C7. Acceptance criteria

- [ ] 5 статей в `lib/blog.ts` (EN + RU тексты, 600–1000 слов EN)
- [ ] `/blog/[slug]` рендерит каждую статью без iframe / Markdown-плагинов
- [ ] DHA-проверка пройдена (`grep -iE 'cure|guaranteed|100%|permanent fix' lib/blog.ts` — пусто)
- [ ] `prose` стили подключены (Tailwind plugin) для нормальной типографики
- [ ] Sitemap включает все 5 url'ов
- [ ] JSON-LD `Article` schema на каждой статье
- [ ] Build чистый, type-check чистый

### C8. Что от пользователя

- ⚠️ **Главное:** одобрение текстов. Каждая статья требует review Анной (как лицензированным специалистом) — иначе DHA Marketing Code нарушаем как личность.
- Ревью на согласованность с её клинической практикой и её tone of voice.
- Опционально: добавить от себя 1–2 конкретных кейса (анонимизированных).

### C9. Оценка трудозатрат

- Написание 5 статей EN (черновик): ~6 часов
- Перевод на RU + локализация формулировок: ~3 часа
- Технические шаблоны (lib/blog.ts, [slug]/page.tsx, BlogArticle.tsx): ~2 часа
- DHA-проверка + ревью + правки: ~2 часа
- **Итого: ~13 часов**

### C10. Альтернатива (вне scope этого ТЗ)

Если статьи писать долго — заменить раздел блога на «Подпишитесь и получите 5 материалов по почте» — лид-магнит вместо публичного блога. Уменьшает SEO, но даёт email-базу для CRM. Решение по этому варианту — после фотосессии (фаза 1).
